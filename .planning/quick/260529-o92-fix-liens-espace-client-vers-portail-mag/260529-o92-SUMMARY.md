# Quick Task 260529-o92 — SUMMARY

## Problème

Cliquer sur « Espace client » dans la vitrine affichait le formulaire de contact
au lieu de l'espace client. Le portail est pourtant déjà développé.

## Cause racine

Dans `resources/views/layouts/app.blade.php`, les trois liens « Espace client »
(header desktop ligne 54, menu mobile ligne 99, footer ligne 151) pointaient vers
`route('contact')` au lieu de l'entrée réelle du portail.

## Correctif

`route('contact')` → `route('portail.magic-link.request')` sur ces 3 liens
uniquement. Cette route (`GET /auth/magic`, guard `guest:clients`, définie dans
`routes/portail.php`) sert le formulaire de demande de magic-link, point d'entrée
correct pour un visiteur non connecté. Une fois connecté, le client atteint
`portail.passages` (timeline des passages).

## Vérification

- Les 3 liens résolvent désormais vers `/auth/magic` (confirmé `php artisan route:list`).
- Les liens `mailto:` / `tel:` du footer sont intacts ; aucun autre `route('contact')`
  des vraies entrées « Contact » n'a été touché.

## Fichiers modifiés

- `resources/views/layouts/app.blade.php` (3 hrefs)

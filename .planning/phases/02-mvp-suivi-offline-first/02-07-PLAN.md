<objective>
Implémenter le **portail client en lecture seule** et l'**authentification magic link** complète au-dessus des fondations posées en 02-01 (guard `clients`, Client extends Authenticatable, package `cesargb/laravel-magiclink` installé, `routes/portail.php` stub vide). Ce plan ferme les requirements **AUTH-02, AUTH-03, AUTH-04, PORT-01, PORT-02** que le plan-checker a identifiés comme non couverts par 02-01..02-06.

**Flow auth (D-49, D-50, D-52, D-54) :**
1. GET `/auth/magic` → formulaire email
2. POST `/auth/magic` → rate limit `magic-link` (5/h IP + 3/24h email), sleep 1-3s, anti-énumération. Si Client existe : `MagicLink::create(LoginAction::guard('clients'), lifetime: 2880, numMaxVisits: 3)`, envoyer `MagicLinkMail` avec URL custom `/auth/confirm?ml={token}` (PAS l'URL `/magiclink/{token}` auto-générée par le package — on substitue pour la page intermédiaire D-50). Message retour générique.
3. GET `/auth/confirm?ml={token}` → vue HTML statique avec `<form method="POST">` + token en hidden. **Pas de validation, pas de consommation** (SafeLinks M365 fait un GET pre-scan, on n'y touche pas — D-50).
4. POST `/auth/confirm` → `MagicLink::getValidMagicLinkByToken($token)` valide expiration + numMaxVisits. Si OK : récupère le Client de l'action, `Auth::guard('clients')->login($client)` (session glissante 30j cookie via `lifetime` du config session), incrémente `num_visits` du magic link, redirige vers `/portail/passages`. Si KO : redirige `/auth/magic` avec flash error.
5. POST `/portail/logout` → `Auth::guard('clients')->logout()` + session invalidate + redirect.

**Portail (PORT-01, PORT-02, D-65) :**
- Livewire 3 `Portail\PassageTimeline` (read-only render-only, copié sur le pattern `GoogleReviews`).
- Eager load `passages.photos` + `passages.piscine`, **filtre strict `where('client_id', Auth::guard('clients')->id())`** — sans ce filtre, fuite inter-clients (RESEARCH §1110, threat T-2-01 transposé).
- Vue Blade transposée 1:1 depuis `mockups/v1/portail.html` : topbar navy + carte piscine drenched + section "Dernier passage" (4-col mesures, chips actions, mot du pisciniste, photos grid) + timeline historique + CTA WhatsApp.
- Lightbox photos en Alpine (swipe touch + flèches keyboard + Escape).

Purpose : sans ce plan, AUTH-02..04 et PORT-01..02 restent fantômes — 02-01 a juste créé les fondations (guard, model, package, route stub) sans aucune surface utilisateur. Le plan-checker a flaggé cette dette ; ce plan la rembourse.

    namespace App\Http\Controllers\Portail;

    use App\Http\Controllers\Controller;
    use App\Http\Requests\MagicLinkRequest;
    use App\Mail\MagicLinkMail;
    use App\Models\Client;
    use Illuminate\Http\Request;
    use Illuminate\Support\Facades\Auth;
    use Illuminate\Support\Facades\Log;
    use Illuminate\Support\Facades\Mail;
    use MagicLink\Actions\LoginAction;
    use MagicLink\MagicLink;

    class MagicLinkController extends Controller
    {
        public function requestForm() { return view('portail.magic-link-request'); }

        public function send(MagicLinkRequest $request)
        {
            // D-52 anti-énumération : sleep aléatoire 1-3s pour uniformiser le timing
            usleep(random_int(1_000_000, 3_000_000));

            $client = Client::where('email', $request->validated('email'))->first();

            if ($client) {
                try {
                    $action = (new LoginAction($client))->guard('clients');
                    $magicLink = MagicLink::create($action, 2880, 3); // D-49 lifetime=48h, numMaxVisits=3

                    // D-50 : URL custom pointant vers NOTRE confirm-view, PAS /magiclink/{token}
                    $token = $magicLink->token ?? null;
                    if (! $token) {
                        // Fallback : extraire depuis ->url si l'attribut diffère
                        $token = \Illuminate\Support\Str::afterLast($magicLink->url, '/');
                    }
                    $customUrl = route('portail.magic-link.confirm-view', ['ml' => $token]);

                    Mail::to($client->email)->send(new MagicLinkMail(
                        magicUrl: $customUrl,
                        clientName: $client->name,
                    ));
                } catch (\Throwable $e) {
                    Log::error('Magic link send failed', ['client_id' => $client->id, 'exception' => $e->getMessage()]);
                    // On NE révèle PAS l'échec — message générique (anti-énumération)
                }
            }

            return back()->with('status', 'Si cet email correspond à un compte, un lien de connexion a été envoyé. Vérifiez votre boîte de réception.');
        }

        public function confirmView(Request $request)
        {
            $token = $request->query('ml', '');
            // D-50 — AUCUNE validation, AUCUN side-effect ici. La page est purement présentationnelle.
            return view('portail.confirm', ['token' => $token]);
        }

        public function confirm(Request $request)
        {
            $token = (string) $request->input('ml', '');
            if ($token === '') {
                return redirect()->route('portail.magic-link.request')->withErrors(['ml' => 'Lien manquant.']);
            }

            $magicLink = MagicLink::getValidMagicLinkByToken($token);
            if (! $magicLink) {
                return redirect()->route('portail.magic-link.request')
                    ->withErrors(['ml' => "Ce lien de connexion n'est plus valide. Il est peut-être expiré ou déjà utilisé. Demandez un nouveau lien."]);
            }

            // Récupérer le Client depuis l'action sérialisée
            // Le package sérialise l'action ; on peut l'invoquer via $magicLink->action->run() qui
            // exécute Auth::guard('clients')->login($user). Si l'API run() retourne une Response/Redirect
            // qu'on veut ignorer (on fait notre propre redirect), on appelle run() puis on ignore son retour.
            try {
                $magicLink->action->run();
            } catch (\Throwable $e) {
                Log::error('Magic link consumption failed', ['exception' => $e->getMessage()]);
                return redirect()->route('portail.magic-link.request')
                    ->withErrors(['ml' => "Une erreur est survenue. Réessayez."]);
            }

            // Incrémenter num_visits manuellement (le package le fait normalement via son middleware
            // sur /magiclink/{token} — comme on bypass cette route, on l'incrémente nous-mêmes)
            $magicLink->num_visits = ($magicLink->num_visits ?? 0) + 1;
            $magicLink->save();

            $request->session()->regenerate();

            return redirect()->route('portail.passages');
        }

        public function logout(Request $request)
        {
            Auth::guard('clients')->logout();
            $request->session()->invalidate();
            $request->session()->regenerateToken();
            return redirect()->route('portail.magic-link.request');
        }
    }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client (browser) → POST /auth/magic | Saisie email non authentifiée — risque énumération + abus envoi mail |
| Email reçu → SafeLinks proxy (Microsoft 365) → GET /auth/confirm | Le proxy pré-scanne le lien avant que l'utilisateur clique. Si GET consomme le token, l'utilisateur arrive sur "lien expiré". |
| Client authentifié 'clients' vs Operator authentifié 'web' | Deux guards cohabitent. Un client NE doit JAMAIS accéder à /admin/*, un operator NE doit JAMAIS apparaître comme client sur /portail/*. |
| Portail Livewire → Eloquent Passage | Une query Passage sans filtre `client_id` fuite les données d'un autre client. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-07A | Information Disclosure | POST /auth/magic — énumération emails | mitigate | Message identique générique "Si cet email correspond…" pour Client trouvé OU absent. `usleep(random_int(1_000_000, 3_000_000))` uniformise le timing. Test M3 (email inconnu n'envoie aucun mail mais retourne le même message). Source D-52, RESEARCH §Pitfall (OWASP Forgot Password Cheatsheet). |
| T-2-07B | Denial of Service | POST /auth/magic — abus envoi mail (cost Brevo) | mitigate | Rate limiter `magic-link` : `Limit::perHour(5)->by(ip)` + `Limit::perDay(3)->by(email)`. Test M4 valide le blocage à la 6e tentative. |
| T-2-07C | Elevation of Privilege | GET /auth/confirm — SafeLinks consomme le token | mitigate | D-50 : la vue `portail.confirm` est PUREMENT statique (aucun side-effect, aucun appel à MagicLink::*). Seul POST consomme. Test M5 vérifie que GET n'incrémente PAS `num_visits`. Source RESEARCH §Pitfall 4, learn.microsoft.com SafeLinks docs. |
| T-2-07D | Tampering | POST /auth/confirm — token forgé | mitigate | `MagicLink::getValidMagicLinkByToken($token)` rejette les tokens invalides (signature HMAC du package). Test M7 valide le rejet d'un token bidon. Si null → redirect avec error générique (pas de leak sur la raison du rejet). |

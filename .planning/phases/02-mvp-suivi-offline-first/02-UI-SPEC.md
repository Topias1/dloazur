# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: MVP Suivi Offline-First.
> Generated by gsd-ui-researcher.
>
> **Source of truth:** `mockups/v1/passage.html` (saisie passage), `mockups/v1/dashboard.html` (shell + liste + badge sync), `mockups/v1/portail.html` (portail client), `mockups/v1/auth.html` (magic link confirm).
>
> **Register:** `product` throughout. The admin back-office is Pierre's working tool; the portal is a trust surface for clients. Neither is brand-forward.

> **Critical constraint:** iOS Safari + Android Chrome, récents et anciens (≥ iOS 14 / Android 5). Plein soleil, une main, réseau hasardeux. Tout ce qui touche à la saisie passage n'utilise pas Livewire — Alpine 3 + IndexedDB uniquement.

| 2 | 8px (0.5rem) | Espacement entre éléments compacts (chips, badges) |
| 3 | 12px (0.75rem) | Padding carte mesure terrain (compact) |
| 4 | 16px (1rem) | Espacement par défaut |
| 5 | 20px (1.25rem) | Padding interne carte mobile standard |
| 6 | 24px (1.5rem) | Padding section, spacing actions liste |
| 7 | 28px (1.75rem) | Padding interne carte desktop |
| 8 | 32px (2rem) | Séparation éléments de layout |

| **13** | **52px (3.25rem)** | **Hauteur bouton primaire "Enregistrer le passage" — touch target critique** |
| **14** | **56px (3.5rem)** | **Steppers terrain (pH, chlore, TAC, sel) — chaque bouton +/−** |
| **15** | **60px (3.75rem)** | **Hauteur topbar (admin + portail)** |
| 16 | 64px (4rem) | Espacement major layout |

**Exceptions non-multiples-de-4 (héritées Phase 1, maintenues) :**
- `spacing-13` (3.25rem = 52px) : hauteur bouton primaire — touch target ≥ 44px.
- `spacing-14` (3.5rem = 56px) : dimensions steppers terrain — touch target ≥ 44px.
- `spacing-15` (3.75rem = 60px) : hauteur topbar.
- Bottom safe-area iOS : `pb-safe` / `padding-bottom: env(safe-area-inset-bottom)` sur la sticky save-bar et la bottom nav mobile — obligatoire pour les iPhone avec home indicator.

**Règle terrain absolue :** Tout élément interactif dans la saisie passage est `h-12` (48px) minimum, steppers `h-14 w-14` (56px). Pas d'exception.

## Typography

Quatre niveaux exacts. Deux poids par fonte : Fredoka 600 + 700 ; Inter 400 + 600. Identique Phase 1.

| Role | Family | Size | Weight | Line Height | Letter Spacing | Usage Phase 2 |
|------|--------|------|--------|-------------|----------------|---------------|
| Headline | Fredoka | `clamp(1.875rem, 3vw, 2.5rem)` | 700 | 1.1 | -0.005em | Titre section dashboard ("Tournée du jour"), titre portail section |
| Title | Fredoka | 1.25rem (20px) | 600 | 1.3 | -0.005em | Titres de section in-passage ("Mesures de l'eau", "Actions menées", "Photos", "Notes"), titres cartes client |
| Body | Inter | 1rem (16px) | 400 | 1.6 | normal | Corps de texte, notes client, descriptions |
| Label | Inter | 0.75rem (12px) | 600 | 1.2 | 0.18em + uppercase | Kickers statut (HORS LIGNE, EN ATTENTE, SYNCHRONISÉ), badge nav |

### Règles spécifiques Phase 2

- **Valeurs mesures (pH, chlore, TAC, sel) dans steppers** : Fredoka 700, `text-xl` (1.25rem), `tabular-nums`, centré dans le puit. Règle "Fredoka pour les mots, Inter pour les chiffres" levée ici par exception mockup — le chiffre est l'élément central visuel d'une carte de mesure.
- **Valeurs mesures dans le portail client** : Fredoka 700, `text-2xl` (1.5rem), `tabular-nums`. Même exception justifiée — preuve visuelle pour le client.
- **Statut mesure** ("idéal", "mg/L") : Inter 400, `text-[11px]`, couleur contextuelle.
- **Notes** (textarea) : Inter 400, 1rem (16px), `leading-relaxed`. Jamais Inter 600 dans un textarea.
- **Nav-label bottom-nav mobile** : Inter 600, 11px — one-off utility `nav-label`, hérité Phase 1, même contexte.
- Toute valeur numérique : `tabular-nums` systématique.
- Pas de texte en dégradé. Emphase par poids ou taille uniquement.

### Règle ambre pour le hors-ligne (critique Phase 2)

Le hors-ligne se dit **toujours en ambre** (`warn` / `warn-bg`). Le rouge (`danger`) est réservé aux erreurs critiques (synchro échouée après 3 retries, erreur API). **Un badge hors-ligne rouge est un bug.**

- Hors-ligne actif : bandeau `warn-bg` + ring `oklch(0.85 0.09 82)` + icône wifi-off `oklch(0.5 0.11 72)` + texte rassurant
- Photo en attente d'upload : badge `warn` sur miniature
- Synchro erreur après 3 retries : badge `danger` sur miniature + bouton "Réessayer" (`azure-500`)
- Passage synchronisé : badge `success` sur miniature

## Interaction Contracts

### Hérités Phase 1 (inchangés)

- Boutons : transition `background-color 200ms cubic-bezier(0.22, 1, 0.36, 1)`. Jamais `hover:scale` sur boutons texte.
- Focus : `focus-visible` → 2px `azure-500` ring, offset 2px.
- `prefers-reduced-motion` : `animation-duration: 0.001ms !important` sur tout.

### Steppers terrain (nouveaux)

- `active:scale-95` sur les boutons +/− : feedback tactile immédiat — exception intentionnelle à la règle "jamais scale" (éléments non-texte, feedback terrain mobile).
- Valeur du puit se met à jour instantanément (Alpine `x-model` ou `@click`).
- Haptic feedback (optionnel, si Vibration API disponible) : `navigator.vibrate?.(10)` sur tap.

### Photos (capture + upload)

- Bouton "Ajouter" déclenche `<input type="file" accept="image/*" capture="environment">` — caméra arrière par défaut.
- Après sélection : spinner inline dans la miniature pendant compression WASM/Canvas.
- Badge statut sur miniature : transition `opacity 150ms` (apparition).
- Drag & drop de photo non requis v1.

### Toast warnings mesures (soft validation)

- Position : `fixed bottom-20 left-4 right-4 z-50` (au-dessus de la save-bar).
- Style : `rounded-2xl bg-warn-bg ring-1 ring-warn/40 px-4 py-3 flex items-center gap-2.5 shadow-md`
- Texte : Inter 600 `text-sm text-[oklch(0.42_0.10_70)]`
- Auto-dismiss 4s (`setTimeout`, Alpine `x-show`).

### Toast erreur synchro 409 (passage déjà clos)

- Position : idem warnings mesures.
- Style : `rounded-2xl bg-danger/10 ring-1 ring-danger/30 px-4 py-3 shadow-md`
- Texte : "Ce passage a déjà été clos. Tes modifications n'ont pas été enregistrées."
- Pas d'auto-dismiss — l'utilisateur doit fermer explicitement (`×`).

### Mise à jour PWA (Update prompt)

- Toast discret en haut de page (pas modal, pas bloquant) : `fixed top-4 left-4 right-4 z-50 max-w-sm mx-auto`
- Style : `rounded-2xl bg-white ring-1 ring-navy-900/8 shadow-md px-4 py-3 flex items-center justify-between`
- Texte : Inter 400 `text-sm text-ink-700` — "Mise à jour disponible"
- Bouton "Recharger" : `h-8 px-3 rounded-xl bg-azure-500 text-white text-sm font-semibold`
- Bouton "Plus tard" : `h-8 px-3 rounded-xl text-ink-500 text-sm` — masque le toast

### Lightbox photos portail

- Ouverture : `opacity 0 → 1` 200ms `ease-out`.
- Fermeture : `opacity 1 → 0` 150ms + `display:none` après.
- Swipe : seuil 50px déplacement horizontal → changement slide.
- Keyboard : `ArrowLeft` / `ArrowRight` / `Escape`.
- Scroll lock sur `<body>` pendant ouverture lightbox.

## Accessibility Contract

Identique Phase 1 — toutes les règles maintenues. Ajouts Phase 2 :

| Règle | Implémentation |

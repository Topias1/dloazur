## Spacing Scale

Tailwind custom spacing tokens (from `theme.js`), plus the standard Tailwind 4/8-point scale.

| Token | Value | Usage |
|-------|-------|-------|
| 1 | 4px (0.25rem) | Icon gaps, inline nudges |
| 2 | 8px (0.5rem) | Tight element spacing, chip padding |
| 3 | 12px (0.75rem) | Compact padding (mobile nav pills) |
| 4 | 16px (1rem) | Default element spacing |
| 5 | 20px (1.25rem) | Card internal padding (compact) |
| 6 | 24px (1.5rem) | Section inner spacing, button padding |
| 7 | 28px (1.75rem) | Card internal padding (standard) |
| 8 | 32px (2rem) | Layout element separation |
| 12 | 48px (3rem) | Sub-section gaps |
| **13** | **52px (3.25rem)** | **Button height (primary) — custom token** |
| 14 | 56px (3.5rem) | Stepper dimensions, large touch targets |
| **15** | **60px (3.75rem)** | **Nav bar height — custom token** |
| 16 | 64px (4rem) | Major layout gaps |
| **18** | **72px (4.5rem)** | **Section vertical padding (custom token)** |
| 20 | 80px (5rem) | Section `py` standard |
| 28 | 112px (7rem) | Section `py` large |

**Exceptions / non-multiples-of-4:**
- `spacing-13` (3.25rem = 52px): button height. Intentional — ≥ 44px touch target + visual balance.
- `spacing-15` (3.75rem = 60px): nav bar height.
- `spacing-18` (4.5rem = 72px): section vertical padding (between standard 16/20px steps).
- `rounded-xl` = 0.875rem (not standard Tailwind): Tailwind 4 default is 0.75rem. Theme overrides to `xl = 0.875rem`, `2xl = 1.25rem`, `3xl = 1.75rem`. These are design tokens, not spacing.
- Hero `pb-28 pt-32`: large hero-specific values, intentional.

## Typography

Four levels exactly. Two weights per font (Fredoka: 600/700; Inter: 400/600).

| Role | Family | Size | Weight | Line Height | Letter Spacing | Usage |
|------|--------|------|--------|-------------|----------------|-------|
| Display | Fredoka | `clamp(2.6rem, 5vw, 4rem)` | 700 | 1.04–1.05 | -0.005em | Hero H1 only — one per page |
| Headline | Fredoka | `clamp(1.875rem, 3vw, 2.5rem)` | 700 | 1.1 | -0.005em | Section H2 titles |
| Title | Fredoka | 1.25rem (20px) | 600 | 1.3 | -0.005em | Card H3, sub-section titles, nav |
| Body | Inter | 1rem (16px) | 400 | 1.6 | normal | Running text, descriptions, footer |
| Small body | Inter | 0.875rem (14px) | 400 | 1.5 | normal | Testimonial attributions, footer nav, form hints |
| Label / kicker | Inter | 0.75rem (12px) | 700 | 1.2 | 0.18em + uppercase | Section kickers above headlines, status badges |
| UI small | Inter | 0.6875rem (11px) | 600 | 1.2 | normal | Bottom-nav labels on mobile only |

**Rules (from DESIGN.md):**
- ≥ 1.25× scale between consecutive levels — no flat hierarchy.
- All numeric values (pH, chlore, dates, totals) use Inter with `tabular-nums`.
- Fredoka for words; Inter for figures and UI controls.
- Body text lines capped at 65–75ch (`max-w-xl` / `max-w-2xl` where needed).
- No text gradients (`background-clip: text`). Color emphasis via weight or size only.

## Interaction Contracts

### Buttons
- Transition: `background-color 200ms cubic-bezier(0.22, 1, 0.36, 1)` (`ease-out-quint`)
- **Never** `hover:scale` or `hover:transform` on buttons — no layout shift
- Active/pressed state on WhatsApp FAB only: `active:scale-95` (mobile tap feedback, intentional exception)
- Focus: `focus-visible` → 2px `azure-500` outline, offset 2px, radius 6px

### Cards
- Service supporting cards: `hover:-translate-y-0.5 hover:shadow-md transition duration-300` — subtle lift
- Gallery photos: `hover:scale-[1.03–1.05] duration-700 ease-out-quint` — slow zoom only
- No `hover:scale` on anything that carries text (layout integrity)

### Hero entrance
- Class `.rise` on hero content: `animation: rise 0.7s ease-out-quint both` (`opacity 0→1`, `translateY 16px→0`)
- `prefers-reduced-motion`: all transitions/animations clamped to `0.001ms`

### Form (Livewire contact)
- Inline validation on blur (Livewire `wire:model.lazy`)
- Submit button shows loading state: spinner replaces arrow icon, button text "Envoi en cours…", disabled
- Honeypot field: hidden from screen readers (`aria-hidden="true"`, `tabindex="-1"`)

### Navigation anchors
- `scroll-behavior: smooth` on `html`
- Active section highlighting in nav: not required for Phase 1 (static markup, Livewire Scroll not needed)

### Auth segmented toggle
- Tab switch: class toggle (`bg-white text-ink-950 shadow-xs` / `text-ink-500`), pane `hidden`/`block` via Alpine.js `x-show`
- No animation needed on tab switch

## Accessibility Contract

Minimum WCAG AA throughout.

| Rule | Implementation |
|------|---------------|
| Touch targets ≥ 44px | All buttons `h-13` (52px) or `h-12` (48px), nav items `h-11` (44px), FAB `h-14 w-14` (56px) |
| Focus visible | `focus-visible` ring 2px `azure-500` on all interactive elements |
| Skip link | `<a href="#top" class="sr-only focus:not-sr-only …">` at top of `<body>` |
| Alt text | All content photos have descriptive French alt text; decorative SVGs `aria-hidden="true"` |
| Color contrast | No light-grey body text; minimum `ink-700` on `sand-50` (ratio ≥ 4.5:1 verified at design time) |
| `prefers-reduced-motion` | `animation-duration: 0.001ms !important` on all `*` in media query (app.css) |
| Heading hierarchy | One H1 per page; H2 for sections; H3 for cards |
| Form labels | All inputs have explicit `<label for="">`, never placeholder-only |
| Disabled nav items | `aria-disabled="true"` + `tabindex="-1"` on greyed sidebar items |
| lang attribute | `<html lang="fr">` on all pages |
| Sunlight legibility | Thème clair par défaut; `ink-950`/`ink-900` on `sand-50` for all body text |

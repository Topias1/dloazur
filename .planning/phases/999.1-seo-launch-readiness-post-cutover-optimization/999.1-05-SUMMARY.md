## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Worktree vendor symlink causes pest rootPath to resolve to main repo**
- **Found during:** Task 2 test run
- **Issue:** `vendor/bin/pest` uses `ClassLoader::getRegisteredLoaders()` to infer `basePath`, which returns the main repo path (vendor is a symlink), not the worktree. This caused `Application::inferBasePath()` to boot the main repo's app — view finder missed the worktree's new `picture.blade.php`, and `public_path()` pointed to the wrong directory.
- **Fix 1:** Added `uses(Tests\TestCase::class)` explicitly in `PictureComponentTest.php` (worktree path extension in Pest.php never fires with symlinked vendor — rootPath always resolves to main repo).
- **Fix 2:** Added `APP_BASE_PATH` env to `phpunit.xml` so `Application::inferBasePath()` returns the worktree root, ensuring the app boots from the correct base.
- **Files modified:** `tests/Feature/Seo/PictureComponentTest.php`, `phpunit.xml`
- **Commits:** 68b8eba

**2. [Rule 3 - Blocker] Artisan command not discovered without autoload refresh**
- **Found during:** Task 1 post-creation verify
- **Issue:** New command class not in composer classmap; `php artisan images:optimize` returned "no commands defined in images namespace".
- **Fix:** `composer dump-autoload -q` refreshes the classmap. In CI (main repo `composer install`), this runs automatically. No code change needed.
- **Impact:** One-time developer action; production and CI unaffected.

<objective>
Ship static-asset image optimization: an idempotent Artisan command (`images:optimize`) that uses `spatie/image` (GD driver) to write `.webp` + `.avif` siblings for the 12 vitrine `.jpg` files, plus a reusable `<x-picture>` Blade component that emits a `<picture>` source set (AVIF → WebP → JPG fallback). Generated siblings are committed to the repo.

Purpose: addresses the audit MEDIUM "serve images as WebP/AVIF". Vitrine images use Blade `asset()` (not Vite imports), so Vite image plugins are the wrong tool (RESEARCH Pitfall 4). The GD path needs no build-env binary (A3). The og:image stays `.jpg` — never swap meta images to next-gen formats.
Output: command + component + generated siblings + a Pest test.

This plan is self-contained (new files only) and runs in Wave 1 in parallel with plans 01 and 02. The `<x-picture>` component is consumed by the new service/city pages in plans 03/04.
</objective>

# Phase 999.1 — UI Design Contract

> Visual and interaction contract for Phase 999.1: SEO launch-readiness & post-cutover optimization.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Register:** brand (vitrine surfaces — all new/modified pages are public marketing pages).
> **Note:** This phase is 95% backend/content work (schema enrichment, new Blade pages, image optimization, sitemap). The UI contract is therefore narrow: it governs the visual treatment of the 4 new service pages, 4 new city-hub pages, the expanded thin pages, and the `<x-picture>` component. It does NOT introduce new interactive components.

## Spacing Scale

All tokens from `resources/css/app.css` `@theme`. Values in rem; pixel equivalents assume 16px root.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (0.25rem) | Icon gaps, inline chip padding |
| sm | 8px (0.5rem) | Compact element spacing, badge padding |
| md | 16px (1rem) | Default inter-element spacing |
| lg | 24px (1.5rem) | Card internal padding, section inner rhythm |
| xl | 32px (2rem) | Between cards in a grid |
| 2xl | 48px (3rem) | Section-level vertical spacing |
| 3xl | 64px (4rem) | Major page section breaks (hero-to-body, etc.) |

Exceptions:
- Touch targets (nav links, CTA buttons, breadcrumb links): minimum 44px height, enforced via `min-h-[2.75rem]` or explicit `h-[3.25rem]`.
- Service/city page hero sections: top padding `pt-16` (64px) + bottom `pb-12` (48px) on mobile; `pt-24`/`pb-16` on desktop — matches existing vitrine page pattern.

## Typography

Source: DESIGN.md §3, `resources/css/app.css` `@theme`. All values are established tokens — do not invent new sizes or weights.

| Role | Size | Weight | Line Height | Font | Notes |
|------|------|--------|-------------|------|-------|
| Display | `clamp(2.6rem, 5vw, 4rem)` | 700 | 1.05 | Fredoka | Hero h1 only; one per page |
| Headline | `clamp(1.875rem, 3vw, 2.5rem)` | 700 | 1.1 | Fredoka | h2 section titles |
| Title | 1.25rem (20px) | 600 | 1.3 | Fredoka | h3 card titles, sub-sections |
| Body | 1rem (16px) | 400 | 1.6 | Inter | All long-form copy; cap lines at 65–75ch |
| Label/Kicker | 0.75rem (12px) | 700 | 1.2 | Inter | `uppercase`, `tracking-[0.18em]`; kickers above section titles |

**Rules (from DESIGN.md):**
- ≥1.25× scale between successive levels. No flat hierarchy.
- All numeric values (pH, chlore, TAC, sel) in Inter with `tabular-nums`.
- Fredoka for words, Inter for numbers/columns.
- Long-form body copy (new service/city pages) must have `max-w-prose` or explicit `max-w-[65ch]`.

## Color

Source: DESIGN.md §2, `resources/css/app.css` `@theme`. All colors are OKLCH tokens. Never use `#000` or `#fff` purs.

| Role | Token | Approx Hex | Usage |
|------|-------|-----------|-------|
| Dominant (60%) | `--color-sand-50` | `oklch(0.987 0.005 85)` | Page background, body surfaces |
| Secondary (30%) | `--color-navy-600` / `navy-800` / `navy-900` | `≈#154c79` / deeper | Hero bands, footer, dark feature cards on service/city pages |
| Accent primary (10%) | `--color-azure-500` | `oklch(0.615 0.211 256)` ≈ `#0080ff` | CTAs, links, active nav, primary buttons |
| Accent rare | `--color-lagon-500` | `oklch(0.720 0.113 207)` | Kicker labels above headlines; "eau saine" badges; step indicators in service page "how it works" |
| Accent rarest | `--color-sun-500` | `oklch(0.760 0.150 72)` | One warm CTA per page max ("Ma piscine est verte ?"); star/rating icons (future) |
| Destructive | `--color-danger` | `oklch(0.620 0.210 25)` | Not used in this phase — no destructive actions |
| Success | `--color-success` | `oklch(0.700 0.150 155)` | "Eau saine" chip in case-study outcomes |
| Text strong | `--color-ink-950` | `oklch(0.255 0.045 250)` | h1/h2/h3 headings |
| Text body | `--color-ink-700` | `oklch(0.445 0.030 250)` | Body paragraphs |
| Text muted | `--color-ink-500` | `oklch(0.585 0.024 250)` | Meta info, breadcrumb trail, secondary labels |

**Accent reserved for (explicit list — never "all interactive elements"):**
- `azure-500`: primary CTA buttons ("Demander un devis", "Nous contacter"), nav active item, hyperlinks in body copy, `<x-picture>` component link borders.
- `lagon-500`: kicker/label text above section titles (e.g., "SERVICE · ENTRETIEN"), step-number circles in protocol lists, city-page `areaServed` accent chips.
- `sun-500`: a single warm CTA button per page maximum (e.g., "Ma piscine est verte ?" on eau-verte page); do NOT use on city hubs.

**Color rules to enforce:**
- Dark hero bands (`navy-900`) must appear on every new service page and every city hub (maintains the engagement rule: "30–60% of at least one strong surface per page").
- `lagon-500` kicker labels (`text-lagon-600`, uppercase Inter 700, 0.75rem, tracking-[0.18em]) appear above every h2 section title on the new pages.
- No bordered left-callouts (anti-pattern from DESIGN.md §6 — use ring, tinted background, or nothing).

## Image Contract (`<x-picture>`)

| Property | Value | Rationale |
|----------|-------|-----------|
| Source formats | `.avif` (first `<source>`), `.webp` (second `<source>`), `.jpg` (img fallback) | Best-to-worst browser support order |
| Generation | Artisan command `OptimizeVitrineImages` (`spatie/image` 3.9.4, GD driver) | No build-env binary dependency (cwebp fallback) |
| Naming convention | `hero-pierre-piscine.jpg` → `hero-pierre-piscine.avif` + `hero-pierre-piscine.webp` (siblings in same dir) | `<x-picture>` strips extension, appends format |
| Loading | `lazy` by default; hero images pass `loading="eager"` | LCP optimization |
| Decoding | `async` always | Non-blocking |
| `alt` | Always required; non-empty for content images | WCAG AA |
| og:image | Stays `.jpg` absolute URL — never next-gen format in meta tags | WhatsApp/Facebook WebP support inconsistent (A2 assumption) |
| Commit strategy | Generated `.webp`/`.avif` committed to repo | Deterministic; no build-env binary needed on Laravel Cloud |

## Accessibility Contract

Source: PRODUCT.md §Accessibility & Inclusion.

| Requirement | Value |
|-------------|-------|
| Contrast | WCAG AA minimum; text on dark hero backgrounds via `sand-50` / `sand-100`; body text is `ink-700` on `sand-50` (verified contrast ratio > 4.5:1) |
| Touch targets | ≥ 44px for all interactive elements (buttons, breadcrumb links, FAQ toggles) |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` clamps all transitions to `0.001ms`. Alpine `x-transition` disabled in reduced-motion. |
| `alt` text | Required on all `<img>` / `<x-picture>` elements; empty `alt=""` for decorative images only |
| Language | `lang="fr"` on root `<html>` (existing); all new copy in French, accents complete |
| Keyboard navigation | FAQ accordion trigger must be `<button>`, not `<div>`. Breadcrumb links are `<a>` with visible `:focus-visible` ring (`azure-500`, 2px, offset 2px). |
| Solar readability | Theme is light-only (dark would be illegible in outdoor sun per PRODUCT.md). No dark-mode toggle added in this phase. |

| browser → PostForm write | Authenticated operator submits title/body/excerpt/slug/cover/status |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-09 | Elevation of Privilege | unauthenticated PostForm write | mitigate | PostForm mounted only inside admin auth-gated views; Livewire requests inherit the auth middleware/session |
| T-06-10 | Tampering | file upload type confusion (non-image / oversized) | mitigate | #[Validate('nullable|image|mimes:jpg,jpeg,png,webp|max:4096')] before any S3 write; medialibrary MIME re-check; PostFormTest asserts non-image rejected |
| T-06-11 | Tampering / XSS | Markdown body rendered publicly via <x-markdown> | mitigate | render path is CommonMark safe_mode (html_input=strip) — already verified by BlogTest 'safe_mode is on'; Pierre is trusted but the public render still strips raw HTML/script; no change to the renderer |
| T-06-12 | Tampering | slug uniqueness collision / injection | mitigate | unique index (Plan 01) + numeric-suffix dedupe in submit(); slug constrained `[a-z0-9-]+` by Str::slug + public route regex |
| T-06-13 | Information Disclosure | livewire-tmp S3 objects lingering | mitigate | livewire:configure-s3-upload-cleanup sets 24h lifecycle (user_setup); temp disk = s3 (no ephemeral local leak) |
| T-06-SC | Tampering | npm install easymde ([ASSUMED] — slopcheck unavailable) | mitigate | blocking-human legitimacy checkpoint before install; verify npmjs.com/package/easymde (no postinstall, canonical repo); never auto-approved |
</threat_model>

<objective>
Build bricks 6 + 7 — the light re-test loop and the carnet local-only — the two retention multipliers delivered at literally zero infra (the directing constraint: infra mini × whaou maxi). The re-test loop is an in-session « As-tu re-teste ? ca a marche ? » prompt (NO push, NO scheduler — V2): a failed re-test routes to the reactive escalation built in Plan 04, capturing the hottest lead (a tried-and-failed user). The carnet (DIAG-07) saves every completed diagnostic on-device (IndexedDB or localStorage) — 0 server, 0 sync, 0 account — with a « Mes diagnostics passes » list readable offline, a « Reprendre ce diagnostic » resume, and an on-device clear. Computing a NEW diagnostic still hits the server (doses stay server-side, DIAG-02); only history reads are local/offline (this is the only relaxation of the SPEC online-only line, scoped to carnet storage per CONTEXT).

This slice grows the Plan 04 wizard view with the re-test prompt + the carnet save/list/resume/clear UI, adds the on-device store JS module, surfaces the returning-visitor « Reprendre mon dernier diagnostic » strip on the landing, and adds a Pest v4 browser test for carnet persistence + offline read.

Purpose: deliver DIAG-07 (carnet local-only) + the reactive half of DIAG-06 (re-test loop feeding escalation) at zero recurring infra.
Output: carnet on-device store + re-test loop + « Mes diagnostics passes » list (S9) + landing resume strip + CarnetLocalTest.
</objective>
